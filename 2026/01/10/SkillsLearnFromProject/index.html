二.防闪屏技术 :
1: while(1) +cleardevice() + sleep( 16 ) + 结束条件 //约60FPS [ 设计简单, 但会存在明显的闪屏 ]
原理 : 清理输出都在前台 + 直接显示
2: 双缓冲技术 : 创建窗口后 + BeginBatchDraw() //进入批量绘制
while( 1 ) {
cleardevice(); // 清空内存后台的缓冲区[ 把旧图去除 ]
//需要实现的图形绘制
FlushBatchDraw(); // 将绘制好的全部内容一次性全部显示
sleep( 16 ) ;
结束条件
}
EndBatchDraw(); //结束批量绘制图片
原理 : 清理输出都在后台 + 一次性输出
3.双缓冲技术 + 时间控制:
在进入批量绘制之后立马记录时间

清理缓存 + 绘图( 时间差 ) + FlushBatchDraw() + 精确计算的sleep() + 结束条件
}

七 : Windows文件操作API ( 对应9 )
#include <io.h> // 主要头文件，包含_findfirst等函数声明
#include <direct.h> // 包含目录操作函数，如_mkdir, _rmdir
Windows中，_findfirst和_findnext是用于遍历目录中文件的标准C函数 , 实现功能 :
查找匹配特定模式的文件（如 .mp3、.txt 等）
获取文件的详细信息（大小、属性、时间等）
递归遍历目录结构
struct _finddata_t fileinfo - 文件信息结构体 ;
内部封装好的 ;
struct _finddata_t {
unsigned attrib; // 文件属性（最重要！）
time_t time_create; // 创建时间
time_t time_access; // 最后访问时间
time_t time_write; // 最后修改时间
_fsize_t size; // 文件大小（字节）
char name[260]; // 文件名（含扩展名）
};
intptr_t handle; //特殊整数类型, 安全储存指针 ;
_findfirst的使用 :
struct _finddata_t fileinfo;
intptr_t handle = _findfirst("C:\Music\*.mp3", &fileinfo); //未找到会返回 -1 ;
fileinfo.attrib & _A_SUBDIR : attrib 是一个无符号整数，每一位代表一个属性 #define _A_SUBDIR 0x10 // 二进制：0001 0000
判断文件第四位是否为1 , 非 0 即文件夹, 是0则是文件
// 在 io.h 中定义的所有属性
#define _A_NORMAL 0x00 // 0000 0000 - 普通文件
#define _A_RDONLY 0x01 // 0000 0001 - 只读文件
#define _A_HIDDEN 0x02 // 0000 0010 - 隐藏文件
#define _A_SYSTEM 0x04 // 0000 0100 - 系统文件
#define _A_VOLID 0x08 // 0000 1000 - 卷标（磁盘名称）
#define _A_SUBDIR 0x10 // 0001 0000 - 子目录
#define _A_ARCH 0x20 // 0010 0000 - 存档文件
